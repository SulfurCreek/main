# 02 — 已驗證 OK 的邏輯 (Verified Logic)

> 這些是經使用者逐案確認「判斷正確」的規則與案例。**改 plan() 後必須用本頁案例回歸測試**，任何一條退化都不可放行。

## ✅ 已確認的規則（使用者點頭過的）

### R1. 辦公室類 vs 專業現場類，鬆緊不同
- 正解屬**辦公室類**（OFFICE_MID：管理幕僚/人資/行政/財會/業務/行銷/法務…）→ 行政支援家族（秘書/行政助理/業務助理/特助…）可寬鬆保留。
- 正解屬**專業技術現場類**（工程/生產/醫療/餐飲/運輸/保全/營建…）→ 行政支援家族**不自動保留**，視為不符。

### R2. function tail 優先於 context keyword
職稱末段是明確職務詞（人資/會計/法務/採購/品管 + 專員/主管/助理/人員），視同部門標籤，公司/前段 context **不覆蓋**它。
- 例：「連鎖通訊門市 人資專員」→ 是人資，不是門市店員。
- 例：「飯店集團管理部 人事主管」→ 是人事，不是飯店職。

### R3. agency 公司：行業 ≠ 職缺
派遣/人力/仲介/顧問公司（萬寶華、才庫、怡高人力、基地人事顧問…）登的是**客戶職缺**，不可用公司「人資/人力」行業去保護人資類項目（除非職稱本身就是人資職）。
- 例：基地人事顧問登「遊戲營運PM」→ 正解產品經理，人事主管要槓。
- 例：怡高人力登「專案經理(人事組)」→ 正解專案經理，人事助理要槓。

### R4. 語言：保護 + 主動補
- **保護**：職稱含特定語言（印尼/越南/泰/日/英/韓/德/法）→ 現有的對應翻譯職類不刪。
- **主動補**：職稱反映語言能力需求（雙語/翻譯/口譯，或 語言+職務角色）但現有沒有翻譯職類 → 主動補到空格。
- **排除**：「X文補習班/教室/家教/老師」（語言是教學科目）、純括號標註「(日文)」（無雙語/翻譯訊號）→ 不補。

### R5. 管理詞保護
職稱含 幹部/主管/領班/組長/課長/店長/經理/主任 → 現有的管理類職類（基層管理幹部、領班/管理幹部、經營管理主管、儲備幹部）受保護。
- 例：「環保清潔幹部」→ 基層管理幹部保留（它是清潔的基層主管）。

### R6. 機構行政共存
公司是機構型（長照/醫療/學校/飯店/集團/中心）→ 行政後勤/管理類項目算機構運作沾邊，可與專業正解共存。
- 例：長照機構「居服組長」→ 行政主管保留。

### R7. 職稱拆解
複合職稱拆成多個語意成分，**每個成分對應的職類都該保護**。
- 例：「居服組長」= 居服（照顧實務指導員 + 居家服務督導員 + 個案管理師都留）+ 組長（行政主管 + 基層管理幹部都留），只槓不相關的人事主管。

### R8. 取代時的連帶槓除（loosely_related）
取代主錯格時，順便槓掉「與正解和職缺領域都不相關的雜項」；但**沾邊的就保留**（擦邊原則）。
- 槓：130327814 環保清潔幹部的「教育訓練人員」（人資類，與清潔完全無關）。
- 留：71696932 工廠儲備幹部只槓「工廠高階主管」，生管/領班/製程都領域相關保留。
- 留：130320309 會計職缺只槓行政助理，電腦操作/總機/檔案這些一般行政保留。

## 三輪自我審查協定

重大規則變動後，使用者要求「分自我審查 3 次再回報」。流程：
1. **第 1 輪**：對問題案例 + 全校準案例跑 plan，列出 mode@格。發現退化 → 修。
2. **第 2 輪**：修正後重跑；用「模擬輸出 5 格」確認實際槓除/建議格位（不只看 mode）。
3. **第 3 輪**：邊界檢查（容易誤傷的相反案例）+ 全量 mode 分布健檢（比例異常代表規則過寬/過嚴）。
全部通過才 apply + 重建北三區 + 回報。
