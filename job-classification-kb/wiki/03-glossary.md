# 03 — 術語表 (Glossary)

| 術語 | 定義 |
|---|---|
| **eNo / 職缺編號** | 職缺的唯一識別碼 |
| **現有1~5** | 職缺目前掛的 5 個職務小類（來源 CSV 的 職務小類1~5 / Excel H~L 欄） |
| **正解 (correct)** | 經語意判斷得出的「單一最貼切職類」，是 plan() 的目標 |
| **建議1~5** | 對照版中，針對該格給出的修正建議（橘色） |
| **葉 (leaf)** | tCode 階層最底層的可選代碼（CodeType 最大值，職務表為 type 3） |
| **中類 (mid / CodeNameB)** | 職類的第二層分類，如「管理幕僚」「人力資源」「醫務護理」 |
| **大類 (CodeNameC)** | 第一層分類，如「管理幕僚／人資／行政」 |
| **辦公室類 (OFFICE_MID)** | 一組「中類」白名單，正解屬此類時可寬鬆保留一般行政項目 |
| **SUPPORT** | 行政支援職類集合（行政助理、秘書、業務助理…），保護規則特別處理 |
| **BROAD_SUPPORT** | SUPPORT 中最泛用、無條件保留的子集（行政助理、行政人員、總機、工讀生、電腦操作） |
| **domain** | 領域規則：依公司名/職稱關鍵字（保全、長照、餐飲…）保護對應職類 |
| **agency 公司** | 派遣／人力仲介／顧問公司，公司行業 ≠ 職缺本身 |
| **function tail** | 職稱末段的職務詞（…人資專員），優先於前段 context（連鎖通訊門市…） |
| **title_has_dept** | 職稱含明確部門標籤（業務部門/財務部…）或 function tail，抑制公司 domain 覆蓋 |
| **loosely_related** | 「沾邊」判斷：某現有項是否與職缺領域相關，決定取代時要不要連帶槓除 |
| **prot (protected)** | 某現有項受保護、不會被槓除 |
| **S / 資科備註** | 資料科學端對「為何不合理」的備註 |
| **T / 企劃備註** | 企劃端寫入的正解判定（「應改為：XXX」「應以「XXX」為主」），是 export 時擷取正解的來源 |
| **ChangeType** | tCode 列的異動類型（新增/改名/合併/UnChange） |
| **CodeType** | tCode 列的階層層級（1=大類, 2=中類, 3=葉，視表而定） |

## plan() 模式速查

| mode | 意義 | 視覺結果 |
|---|---|---|
| `keep` | 不動 | 5 格原樣，無建議 |
| `keep_strike` | 正解已在某格，槓掉其他錯誤格 | 錯誤格刪除線，正解格不變 |
| `replace` | 槓 1 格、放正解 | 1 格刪除線+橘色建議 |
| `replace_multi` | 槓主錯格+不相關雜項 | 多格刪除線，主格有橘色建議 |
| `replace_all` | 5 格全錯全槓 | 全刪除線，建議1=正解 |
| `add` | 有空格直接補 | 空格出現橘色建議，現有不動 |
