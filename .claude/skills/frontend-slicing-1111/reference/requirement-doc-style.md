# 給工程師的需求文件 寫法

骨架用求才 template（搭 `spec-doc-1111` skill），但**內容寫法**遵守下面這套 —— 重點：給人看、不是給 AI 看。

## 原則

1. **白話**：像同事在交辦，不要生硬條列。
   - ✗「標題放大、改用設計系統標題級距與中性主色」
   - ✓「把頁面大標『信件訊息』放大、換成設計稿指定的字：28px、中等粗、深色 #212529」
2. **逐一對應一個 HTML 標籤/區塊**為一節。該標籤若有 CSS 改動，就把「改哪個檔、第幾行」寫在同一節。
3. **值用反引號**：`#1a66ff`、`16px`、`.isNoWish`。
4. **結構前提用明確指示，不要開放式提問**。
   - ✗「請工程師確認要採哪種版面…」
   - ✓「把整塊 `#bookmark.titleBar` 從 `.whiteBg.list > .msgList` 內搬出來，移到 `.whiteBg.filter` 正上方」，附「搬之前/搬之後」位置對照。
5. **CSS 行號**標明是「隨附 mock 檔」的，正式 codebase 要用 selector 對。

## 標記符號（一眼分辨重點）

- 🔧 = 這節要動到 HTML（其餘都只改 CSS）。
- ✅ = mock 已照正式結構做好，照著抄就對。
- 🚧 / 紅字 = 要特別留意、容易踩雷、或與正式環境不同處。
- 維持原值的（顏色不變等）也要寫出來，讓工程師知道「這個不要動」。

## 一節的範本

```markdown
## N 區塊名稱 `<tag class="...">`（白話描述）

把 OO 改成 XX（一句講清楚要的結果）。

* **要改成**：...（值用反引號）
* **原本**：...
* **改哪**：`resumePoolNoticeMail.css` 第 1686–1693 行（新增的 `.headingBar .Title` 規則）
* **HTML**：不用動。
```

## 必備章節

- 版控紀錄（每次發布加一列，格式 `[異動區段] 內容`，版號 `主.次.修`）。
- 「怎麼讀這份文件」前言：標清楚「只有第 X 節要動 HTML，其餘只改 CSS」「字體一律微軟正黑體」「離線替身要排除」。
- 附錄「CSS 行號速查」表。
- 「不需處理/排除」一節：離線替身、mock 補值。

## 意願這類「結構＋顏色」需求的寫法

先給**結構**（附 HTML 範例），再給**顏色**，最後標 mock 狀態（✅ 已照此結構 / 🔧 別照舊 mock）。例見本任務 `需求文件` §11。
